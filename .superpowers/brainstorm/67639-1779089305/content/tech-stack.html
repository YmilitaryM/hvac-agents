多智能体框架选型 — 方案对比

HVAC制冷站多智能体系统的技术栈推荐

A

LangGraph + 自定义仿真 (推荐)

LangGraph 做多智能体状态编排，自定义Python模型做HVAC设备仿真，FastAPI + MQTT做数据层

优势

  • 状态图模型天然适合制冷站控制流程（启停、加减载、模式切换）
  • 灵活的Agent间通信和工具调用
  • 生态成熟，与LLM集成方便
  • 可控的复杂度，不会过度抽象

劣势

  • 需要自己设计Agent拓扑和通信协议
  • LangGraph学习曲线中等
B

CrewAI + 自定义仿真

CrewAI 做角色化多智能体协作，自定义Python模型做HVAC设备仿真

优势

  • 开箱即用的角色分工（Crew/Agent/Task）
  • 上手快，概念直观

劣势

  • 实时控制和状态管理能力较弱
  • 框架对执行流程的控制不如状态图精细
  • 与工业控制场景的契合度不如LangGraph
C

纯 Python + asyncio + 自建框架

不用任何Agent框架，完全自建多智能体通信和状态管理

优势

  • 最大灵活性，零框架约束
  • 轻量，无额外依赖
  • 完全掌控执行逻辑

劣势

  • 需要自行实现大量基础能力（状态管理、Agent通信、错误恢复）
  • 开发周期长
  • 后期维护成本高

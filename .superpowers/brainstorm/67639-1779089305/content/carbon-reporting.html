Q1: 碳排放专家 — 独立还是合并？

能源专家关注的是 省钱（电价峰谷、需量管理），碳排放专家关注的是 合规与减碳（排放核算、碳配额、碳交易）。

推荐：拆分为两个独立专家，因为它们的优化目标经常冲突。

例如：峰谷套利建议低谷多开机组蓄冷 → 但此时电网碳排放因子更高（煤电基荷）→ 两个专家给相反建议

维度能源专家碳排放专家
核心目标最小化能源成本最小化碳排放 + 合规达标
知识体系电价政策、需量费率、电力市场规则GHG Protocol、ISO 14064、碳配额、CCER
关键指标单位冷量成本 (元/RT·h)、总电费碳排放强度 (kgCO₂/RT·h)、配额余量
优化手段峰谷套利、需量控制、设备组合优化清洁能源时段优先、碳配额交易建议

Q2: 报告生成 — 专门的报告Agent

新增：📊 报告Agent

不参与决策辩论，而是消费其他Agent的输出，生成多类型报告。

碳排放报告
  • 碳排放总量与强度趋势
  • 碳配额使用进度
  • 减排措施效果评估
  • 合规达标状态
能效报告
  • 系统COP/EER趋势
  • 各设备能效对比
  • 能耗异常分析
  • 节能措施效果
运行报告
  • 设备运行时长与启停统计
  • 异常事件汇总
  • 策略执行效果评估
  • 维护建议
报告生成流程 (LLM驱动)
1. 数据聚合 → 从时序库+事件库拉取原始数据
2. 指标计算 → 按行业标准公式计算KPIs
3. 趋势分析 → LLM分析数据趋势、发现异常模式
4. 结论生成 → LLM基于模板+数据生成叙述性分析
5. 建议生成 → 调用领域专家Agent获取针对性建议
6. 多格式输出 → Markdown/PDF/Excel/看板

Q3: 行业标准查询 — 共享RAG知识库

不是新增Agent，而是共享工具 + 知识库

所有专家Agent通过同一个 RAG 工具查询标准，在辩论时引用标准作为论据。

标准类型具体标准主要使用者
暖通设计GB 50736、ASHRAE 90.1、GB 50189暖通专家
机电设备GB 19577 (冷水机组能效)、GB 18613 (电机能效)机电专家
碳排放ISO 14064、GHG Protocol、GB/T 32150碳排放专家
能源管理GB/T 23331 (ISO 50001)、GB 17167 (能源计量)能源专家
碳排放核算企业温室气体排放核算方法与报告指南、电网排放因子碳排放专家、报告Agent
运行管理GB/T 17981 (空调通风系统经济运行)、JGJ/T 348策略Agent、参数Agent
技术实现： 标准文档 → PDF解析/分段 → Embedding → 向量库 → 各Agent通过 RAG Tool 查询
查询示例： "GB 50736 中关于制冷站能效限值的规定是什么？"

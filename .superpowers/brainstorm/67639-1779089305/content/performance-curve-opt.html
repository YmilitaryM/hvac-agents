基于性能曲线的联合优化 — 策略与参数一体化

设备性能曲线 + 系统合成模型 → 联合优化 → LLM选择 → 执行

核心问题：为什么不能先选设备再调参数？

制冷站是一个强耦合系统。冷却塔风扇转速影响冷却水温 → 冷却水温影响冷机COP → 冷机COP影响最优负荷分配 → 负荷分配影响水泵频率。这些变量不能拆开优化。

系统性能曲面 — 不是单条曲线，是多维曲面
维度独立变量耦合关系
冷机 负荷率、冷冻水出口设定值 COP = f(负荷率, 冷冻水温, 冷却水温)。冷却水温又取决于冷却塔
冷却塔 风扇频率、进水温度、水量 逼近度 = f(风扇频率, 水量, 室外湿球温度)。出水温直接决定冷机冷凝温度
水泵 频率、阀门开度 流量影响冷机蒸发器换热效率，也影响冷却塔效率
系统级 设备组合 + 负荷分配 总功率 = Σ冷机功率 + Σ泵功率 + Σ塔功率。三者通过热力/水力耦合
举例：为什么不能分开算
目标负荷500RT。方案A: 1台冷机100%负荷，冷却塔风扇100%。方案B: 2台冷机各50%负荷，冷却塔风扇60%。
方案A冷机COP高(单机高效区)，但冷却塔风扇全速耗电大。
方案B冷却塔省电，但2台冷机都在低效区，且2台泵运行增加了泵功率。
不联合计算系统总功率，就无法判断哪个更优。

Pydantic Schema
MINLP求解器 输入：设备性能曲线(多项式拟合) + 当前约束 + 预测负荷 + 电价/碳价
MINLP求解器 输出：3-5个帕累托最优方案 {设备组合, 每台设备参数, 系统COP, 成本, 碳排, 可靠性余量}
LLM (策略Agent) 的job：从帕累托解中选一个，给出为什么选这个的推理——考虑天气不确定性、设备健康趋势、操作员偏好

设备性能曲线 — 从哪里来，如何维护

阶段曲线来源精度更新方式
仿真阶段 (初期) 设备铭牌参数 + 厂家样本曲线 → 多项式拟合 中等 手动配置
实测校准 (后期) 实际运行数据回归 → 在线更新曲线参数 高 自动周期性重拟合
退化追踪 (持续) 曲线偏移量 = 实际COP - 预期COP → 量化性能衰减 退化趋势 监测Agent提供退化信号
冷机COP曲线的数学表示 (多项式拟合)：
COP(PLR, Tchw, Tcw) = a₀ + a₁·PLR + a₂·PLR² + a₃·Tchw + a₄·Tcw + a₅·Tcw² + a₆·PLR·Tcw
PLR = 部分负荷率 (0~1), Tchw = 冷冻水出口温度, Tcw = 冷却水入口温度
系数 [a₀..a₆] 由厂家数据或运行数据回归得到，定期更新

这对 Agent 设计意味着什么？

变化之前现在
策略Agent LLM推理 + 简单枚举设备组合 调用联合优化引擎获取帕累托解集 → LLM从解集中做有依据的选择
参数Agent 独立做连续参数优化 角色变化：不再独立做优化决策，变为实时微调控制器
在策略执行期间，根据实时负荷波动做±5%范围内的小幅调整
新增 — 联合优化引擎 (非Agent，是数值计算服务)
MINLP求解 + 帕累托前沿 + 性能曲线管理

参数Agent的新定位：

策略Agent输出的方案包含了每个设备的目标参数（如冷机出水7°C，泵频率40Hz）。参数Agent在策略执行期间实时监控，当实际负荷偏离预测值在±5%以内时，自动微调参数（如将出水调为6.8°C或7.2°C）而不需要重新走策略生成流程。如果偏差超过5%，则触发新的策略生成循环。

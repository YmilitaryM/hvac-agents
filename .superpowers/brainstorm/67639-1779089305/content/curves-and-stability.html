性能曲线获取 & 系统稳定性设计

不用厂家曲线 — 用仿真+数据回归 | 策略必须包含过渡过程

Q1: 不用厂家曲线，用什么？— 三层曲线获取策略

阶段方法具体做法
Layer 1
仿真初始化		 物理机理模型 • 冷机：逆卡诺循环 + 效率修正系数 → COP = f(PLR, Tevap, Tcond)
• 冷却塔：Merkel理论 + 传热单元数法 → 逼近度 = f(风量, 水量, 湿球温度)
• 水泵：相似定律 → 功率 = f(频率, 流量, 扬程)
• 关键：这些模型需要的参数（换热面积、压缩机效率等）从典型设备参数推算，不需要具体厂家的专有数据
Layer 2
仿真→优化		 代理模型 (Surrogate Model) • 用物理模型跑大量仿真工况（覆盖负荷率10%~100%、各种温度组合）
• 生成合成数据集 → 多项式回归 → 得到轻量的COP曲面方程
• 优化引擎用的就是这些多项式代理模型，不是物理模型（物理模型太慢）
Layer 3
实测校准		 在线参数辨识 • 接入真实设备后，用运行数据在线修正代理模型的系数
• 方法：递推最小二乘(RLS) 或 卡尔曼滤波 → 系数随设备老化缓慢漂移
• 监测Agent提供退化信号，当实际COP偏离模型预测超过阈值 → 触发重拟合
• 不依赖任何厂家数据，完全自适应

三层曲线获取 — 数据流

物理机理模型 (仿真层，运行一次)
  → 产生大量工况-性能数据点 (离线)
  → 多项式回归拟合 → 轻量代理模型 (优化引擎使用)

代理模型 (优化引擎运行时)
  → 每次优化时读取当前的代理模型系数
  → 计算速度: 微秒级 (就是多项式求值)

在线参数辨识 (实测运行后持续)
  → 实时数据流 → RLS/KF更新系数
  → 当 ||实际COP - 预测COP|| > 阈值 → 通知监测Agent
  → 监测Agent标记"设备性能异常"或"模型需重标定"

Q2: 超调与稳定性 — 策略不能只有终态，必须有过渡过程

问题本质：制冷站有巨大的热惯性。冷冻水系统有数百吨水，建筑物有热容。一个"15:00停2号冷机"的指令如果直接下发，会导致出水温度剧烈波动、系统压力冲击、甚至触发其他设备的保护停机。

6. 预期效果：COP、节能、碳减排

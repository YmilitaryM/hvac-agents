标准如何在报告中引用

报告Agent在Phase 2和Phase 4两次查询RAG标准库：

Phase 2 查询："GB 19577中冷水机组COP的限定值和节能评价值分别是多少？" → 得到具体数值，用于对标计算
Phase 4 查询："ISO 14064关于Scope 2排放的报告格式要求是什么？" → 引用标准条款编号和原文，嵌入报告

3. 能效对标分析

本期系统COP为5.2，高于GB 19577-2024规定的冷水机组能效限定值( COP≥4.0 )，但低于节能评价值( COP≥5.6 )。与上一统计周期(COP 4.9)相比提升6.1%，主要得益于15:00减机策略实施后单机负荷率从58%提升至75%。

报告Agent的内部结构

组件职责技术
模板库每种报告的结构化模板（章节、指标定义、图表占位符）Jinja2 / Markdown模板
KPI计算器按标准公式计算关键指标（COP、EER、碳排放强度等）Python函数库，单元测试覆盖
数据分析引擎趋势识别、异常检测、同比环比、对标分析Quick LLM + 统计方法
标准引用器查询RAG标准库，获取条款编号和原文向量检索 + LLM精排
叙述生成器将数据和分析转化为自然语言段落Quick LLM + 结构化Prompt
格式渲染器Markdown → PDF/Excel/JSONWeasyPrint / Pandoc / openpyxl
调度器接口注册定时任务，响应按需查询与L4调度引擎对接

报告Agent与决策Agent的关系

报告Agent是只读消费者，不参与决策闭环

决策闭环 (写入):
监测 → 预测 → 策略 → Advocate辩论 → 协调 → 安全 → 参数 → 执行
(Report Agent 不在此链路上)

报告闭环 (只读):
所有Agent输出 → 📊报告Agent → 报告输出
(Report Agent 消费所有Agent的结构化输出，但不回写)
